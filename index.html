What is Ansible Automation Platform?

Red Hat's enterprise platform for building, running, and managing IT automation at scale. Combines multiple open-source Ansible tools into one supported product used by thousands of enterprises.

Core Components

  • Automation Controller Run and schedule playbooks at scale with role-based access control (RBAC)
  • Automation Hub Share, manage, and consume certified Ansible collections across the organization
  • Event-Driven Ansible Trigger automation from IT events automatically without manual intervention
  • Ansible Lightspeed AI-powered playbook generation from natural language descriptions

Enterprise Value

  • Red Hat packages, tests, certifies, and supports it so customers get stability and security patches

Ansible Automation Portal: 6 Backstage Plugins

A self-service web UI built on Red Hat Developer Hub (Backstage framework). Gives teams a single place to discover collections, run job templates, and manage automation without CLI expertise.

1
Catalog Provider
Syncs resources from AAP Controller and Hub into the Backstage catalog. Enables Backstage to discover and track job templates, collections, and execution environments automatically. Runs periodic sync on configurable intervals.
2
Scaffolder Actions
Provides golden-path templates for provisioning new automation projects. Teams create collections, roles, and EEs from pre-approved templates. Integrates with the Backstage software template engine for guided workflows.
3
Job Templates
Browse, launch, and monitor AAP job template runs directly from the portal. Includes real-time sync that refreshes the template list when new templates are added to the AAP Controller. Supports search, filtering, and starring.
4
Collections
Search and explore Ansible content collections from Automation Hub. Server-side pagination handles large catalogs efficiently, loading only ~6KB per page instead of the full dataset.
5
Self-Service UI
Main frontend with dashboard, EE builder, Git repositories, and template cards. Permission-gated sidebar items conditionally render based on RBAC policies. Includes Settings page for portal configuration.
6
Common (Shared)
Shared API client, authentication, RBAC permissions, and SCM client. Every other plugin depends on this for secure communication with AAP backend services.

The Portal: Architecture & Deployment

Built on Red Hat Developer Hub (RHDH), the enterprise distribution of Backstage by Red Hat. Gives teams a single place to discover collections, launch job templates, and manage automation. Open source upstream: github.com/ansible/ansible-backstage-plugins

Execution Environment (EE) Builder

A core portal feature that lets users build custom Ansible Execution Environments from the UI without writing Dockerfiles.

Visual EE Definition
Select base image, add Python packages, system packages, and Ansible collections through a guided form.
Build & Push
Generates execution-environment.yml, builds the container image using ansible-builder, and pushes to a configured registry.
Live Build Logs
Real-time streaming build output in the portal. Users can track image build progress, debug failures, and verify successful pushes.

My Contribution: MCP Tools for the Portal

Built a Backstage plugin (ansible-mcp-tool) that exposes portal data to AI assistants via the Model Context Protocol (MCP). All 6 tools authored by me.

Why MCP?

1. Data locked in the UI
The portal holds collections, templates, and EEs but this data is only accessible through the web interface. AI assistants cannot reach it.

2. MCP bridges the gap
MCP (Model Context Protocol) is an open standard that lets AI tools call server-side functions over JSON-RPC 2.0, running inside the RHDH backend.

3. Works with any AI client
Claude Code, Cursor, Copilot, or any MCP-compatible client can now query, analyze, and manage portal resources directly from the IDE.

generate-ansible-ee-definitionGive it a playbook, get a ready-to-build EE YAML
analyze-ansible-playbookGap analysis: available vs missing collections
search-collections-by-capabilitySemantic search by what collections do
import-template-from-githubRegister a template into the portal catalog

VS Code Ansible Extension

A VS Code extension by Red Hat that turns VS Code into a full-featured Ansible IDE. Open source: github.com/ansible/vscode-ansible

Core IDE Features

  • Syntax highlighting for YAML playbooks and Jinja2 templates
  • Real-time linting with ansible-lint and yamllint integration
  • Smart auto-completion for module names, parameters, and values
  • Hover documentation shows module docs without leaving the editor
  • Go-to-definition jumps to module and role source code
  • Content scaffolding creates collections, roles, and EEs from VS Code

AI & Generative Features

  • Ansible Lightspeed provides AI-powered code generation from natural language
  • Generate Playbook and Explain Playbook from natural language prompts
  • Generate Role and Explain Role with AI assistance
  • Interactive chat lets you ask Ansible questions directly in VS Code

Supported AI Providers

  • WCA: IBM watsonx Code Assistant
  • Google Gemini: Direct integration
  • Red Hat AI / BYOLLM: Bring Your Own LLM, connect your own model endpoint

Ansieyes: AI-Powered GitHub Bot

A GitHub App powered by Gemini AI that automates issue triage and PR review. Improved by me.
Triggered by commenting \ansieyes on any issue or pull request.

Issue Capabilities

  • AI Issue Triage: Classifies type, severity, finds root cause, suggests fix
  • Two-pass architecture: Librarian finds relevant files, then Surgeon does deep analysis
  • Duplicate Issue Check: Detects and flags duplicate issues before triage
  • Prompt Injection Check: Detects and blocks prompt injection attempts in issues
  • Jira Ticket Creation: Automatically creates a Jira ticket from the triaged issue with all context

Pull Request Capabilities

  • Interactive Menu: Comment \ansieyes on a PR to get a numbered menu
  • 1 Code Review: AI reviews for bugs, security issues, and error handling
  • 2 PR Walkthrough: Executive summary + Mermaid sequence diagram
  • 3 Auto-Fix: Generates one-click GitHub suggestion blocks
  • 4 Full Analysis: Combines review, walkthrough, and auto-fix
  • _ask Query: Ask any free-form question about a PR or issue with full context

Also: auto commit summary on every push, prompt injection protection, per-repo configs
